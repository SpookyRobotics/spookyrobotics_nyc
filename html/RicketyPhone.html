RicketyPhone

Repurposing ancient technology

As a smartphone user who values being on the cutting edge, I have consistently replaced my phone every two years. This has lead to a pile of devices that have sat in my drawer for years. I knew that one day I would want to use these miniature computers for side projects. RicketyPhone is a collection of libraries that allow an older Android phone to be used as a display and interface to a single board computer.

Bill of Materials

  • Android phone running Api 9+
  • Raspberry Pi Zero
  • Micro USB to USB Type A adapter
  • Micro USB cord
  • Power Supply for Raspberry Pi

Methodology

The phones have various sized screens with varying damage to them. The solution must be able to disable parts of the screen for output. Will need to build a way for a connected computer to communicate with the phone for drawing. The phone will need to run an application that defines its working area, and display visual input as best as possible. We will be using the GraphPaper Experiment to draw a bitstream of information outputted by the Raspberry Pi.

My oldest phone is API 9. After researching api capabilities, I read I will have to communicate over USB with the phone. The phone i have doesn't have the accessory support library, so I will use the ADB running on raspbian to communicate with the phone. Any application on the Pi will need to use ADB commands to communicate with the phone. The application will send an intent with any relevant data. The phone will run an application that listens for these intents, and displays the relevant data.

To limit the drawing area, will write an Android application that has a custom layout per phone. An attached device will be able to request a graphic be drawn in a section of the screen. There is a limit between how much data can be sent from an intent to an app, so we must be sure to stay within those limits.

Use Cases

  1. Sensor connected to Raspberry Pi. Sensor sends value to Pi. Pi wants to display value on Phone. Pi sends intent over ADB and phone displays value. Raspberry Pi now has a visual display.
  2. Raspberry Pi wants to read a value from Phone. Pi sends intent to Phone signaling requested information. Phone can a) send intent with response or b) write response to local storage for reading by Pi over ADB. Raspberry Pi now has all sensors on phone.
